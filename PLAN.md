Основание: [PROJECT.md](./PROJECT.md)

| Компонент | Статус |
|-----------|--------|

```
URL → парсинг статьи → промпт для действия → OpenRouter → результат в UI
```

## Порядок действий

### Шаг 1. Вынести подготовку текста статьи в общую функцию

**Файл:** `lib/openrouter.ts`

1. Выделить функцию `formatArticleForPrompt(article)` — собирает `title`, `date`, `content` в единый текст (как в `translateArticleText`).
2. Применить ограничение длины (`MAX_CONTENT_LENGTH`) один раз для всех действий.
3. Проверить, что при пустом `content` возвращается понятная ошибка.

**Зачем:** три новые кнопки используют один и тот же вход — распарсенную статью.

---

### Шаг 2. Добавить AI-функцию «О чем статья?»

**Файл:** `lib/openrouter.ts`  
**Действие:** `summary`

1. Создать `summarizeArticle(article)`.
2. System prompt: профессиональный редактор, отвечает на русском.
3. User prompt: кратко (3–5 предложений) объяснить, о чём статья; указать тему, ключевую мысль и для кого материал.
4. Параметры: `temperature: 0.3`, модель `deepseek/deepseek-chat`.
5. Вернуть только текст ответа без пояснений.

**Ожидаемый результат:** связный абзац на русском в блоке «Результат».

---

### Шаг 3. Добавить AI-функцию «Тезисы»

**Файл:** `lib/openrouter.ts`  
**Действие:** `theses`

1. Создать `extractTheses(article)`.
2. System prompt: аналитик, отвечает на русском.
3. User prompt: выделить 5–10 ключевых тезисов в виде маркированного списка; каждый тезис — одна ёмкая мысль.
4. Формат ответа: `- тезис` (markdown-список).
5. Температура: `0.2` (более детерминированный вывод).

**Ожидаемый результат:** маркированный список тезисов на русском.

---

### Шаг 4. Добавить AI-функцию «Пост для Telegram»

**Файл:** `lib/openrouter.ts`  
**Действие:** `telegram`

1. Создать `generateTelegramPost(article)`.
2. System prompt: SMM-редактор, пишет на русском для Telegram-канала.
3. User prompt:
   - цепляющий заголовок (1 строка);
   - краткий лид (2–3 предложения);
   - 3–5 буллетов с главным;
   - призыв к действию или вопрос аудитории;
   - 2–4 релевантных хэштега в конце.
4. Ограничить длину поста (~1500 символов).
5. Температура: `0.5` (чуть живее, чем у тезисов).

**Ожидаемый результат:** готовый к публикации пост в Telegram.

---

### Шаг 5. Подключить функции в API

**Файл:** `app/api/process/route.ts`

1. После `fetchAndParseArticle(url)` добавить ветку `switch (action)`:
   - `summary` → `summarizeArticle(article)`
   - `theses` → `extractTheses(article)`
   - `telegram` → `generateTelegramPost(article)`
   - `translate` → `translateArticleText(article)` (уже есть)
2. Убрать заглушку с `JSON.stringify(article)` для этих действий.
3. В ответе API возвращать `{ result, action, article }` — формат как у перевода.

---

### Шаг 6. Проверить UI

**Файл:** `app/components/ArticleProcessor.tsx`

1. Убедиться, что для `summary`, `theses`, `telegram` результат показывается как текст (не JSON), как у `translate`.
2. Проверить подписи загрузки из `ACTION_LOADING_LABELS` — уже заданы.
3. При необходимости: для тезисов и поста использовать `whitespace-pre-wrap`, чтобы сохранить переносы строк.

---

### Шаг 7. Настроить окружение и деплой

1. Локально: `OPENROUTER_API_KEY` в `.env.local`.
2. На Vercel: **Settings → Environment Variables** → `OPENROUTER_API_KEY` для Production / Preview / Development.
3. После добавления переменных — **Redeploy**.

---

### Шаг 8. Тестирование

Для каждой кнопки проверить на 2–3 реальных URL:

| Кнопка | Что проверить |
|--------|---------------|
| О чем статья? | Краткий связный ответ на русском, без «воды» |
| Тезисы | Список из 5–10 пунктов, без дублирования |
| Пост для Telegram | Читаемый пост, укладывается в лимит Telegram |

Также проверить:

- некорректный URL → ошибка 400/422;
- сайт без контента → понятное сообщение;
- отсутствие API-ключа → понятная ошибка;
- длинная статья → обрезка + корректный ответ.
